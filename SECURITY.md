# Security policy

## Supported versions

Only the latest tagged public-beta release receives security fixes.

## Reporting a vulnerability

Do not open a public issue containing credentials, LINE IDs, signed media URLs,
private report content, or an unpatched exploit. Use GitHub's private
vulnerability reporting for this repository. If that feature is unavailable,
open a minimal issue asking the maintainer for a private contact channel.

Include the affected version, impact, reproduction steps, and whether secrets
or user data may have been exposed. You should receive an acknowledgement
within seven days. No bounty is currently offered.
